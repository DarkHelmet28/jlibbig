Language syntax and semantic:

General Structure:
Strings can have three section, only appearing in this order:
  <signature section>
  <reaction rules section>
  <bigraphs section>
Every section can also be empty, but if a bigraph with controls is defined, then its controls must appear in the signature section.

Signature:
In the signature section, controls must be defined as it follow:
  %active name : num ;
  %passive name : num ;
where name is the name of the control and num is the number of its ports.

Reaction rule: Reaction rules are defined as it follow:
  redex -> reactum ;
where redex and reactum are two bigraph.

Bigraphs in the bigraph section are separated by semicolon.
Syntax and semantic of bigraph , redex and reactum are the same.

Bigraph grammar:
  B ::= B || B
  B ::= B | B
  B ::= B.B
  B ::= <name/nameslist>
  B ::= <name/nameslist> B
  B ::= C[names]
  B ::= $integer
  B ::= 0
  B ::= 1
  B ::= nil
  B ::= ( B )

  C ::= control's name

  nameslist ::= name , nameslist
  nameslist ::= name

  name ::= [a-zA-Z][a-zA-Z0-9]
  name ::= -
In addition:
names can be empty.
In the occurrences of <name/names>, name can be empty.
Every name can be preceded by + or -, indicating if they refer to a inner or outername. names without + or - will be treated as outernames.

Bigraph operations:
The following list is ordered according to the operators' precedence, from highest to lowest.

  • . : composition of two bigraphs. The inner face of the first bigraph will be linked with the outerface of the second bigraph. If some names of the first bigraph's innerface doesn't occur in the second bigraph's outerface, then they will be added and will appear in the composed bigraph's innerface. The same applies to the second's bigraph outerface.
  • <name/nameslist> : followed by a bigraph, this construct will modify the bigraph's link graph. name (if not empty) is an outername and will appear in the resulting bigraph's outerface. This outername will be linked with every name in the nameslist. If name is empty, names in the list will be linked together with a new edge. Names in the list will be then composed with bigraph's outerface, with the same rules of composition. If <name/nameslist> isn't followed by a bigraph, it represent the resulting bigraph of the operation <name/nameslist> 0 (where 0 is the empty bigraph).
  • | : juxtaposition of two bigraphs, followed by a merging of all roots (the resulting bigraph will only have one root). if bigraphs have some outernames in common, they will be connected in the resulting bigraph.
  • || : juxtaposition of two bigraphs (without merging roots).
